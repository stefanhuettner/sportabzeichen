# Berichts-Workflow

## Notion = Wissensspeicher
- Nachtschicht-Berichte (komplett mit Dokumenten als Unterseiten)
- To-Do-Datenbank (alle Aufgaben, erledigte bleiben sichtbar)

## Telegram = Täglicher Dialog
- 06:00 Morgen-Briefing (ALLE Kalender-Termine inkl. PSBK, Top 3, Nachtschicht-Zusammenfassung)
- 20:00 Abend-Report (Tagesrückblick, Motivation, Nachtschicht-Plan, Änderungswünsche?)

## Slack = Projekt-Dashboard + Arbeitslog
- Jeder Projekt-Channel hat GEPINNTE STATUS-NACHRICHT → regelmäßig aktualisieren
- Struktur: ✅ Fertig / 🔄 In Arbeit / ⏳ Wartet auf Stefan / ⏭️ Nächste Schritte
- Bei fertiggestellter Arbeit → kurze Info im Channel
- Stefan öffnet Slack → sieht sofort pro Projekt den Stand

## Nach jeder Nachtschicht
1. Bericht in Notion anlegen (Hauptseite + Dokumente)
2. Gepinnte Slack-Nachrichten aktualisieren
3. Notion To-Dos aktualisieren
4. 06:00 Telegram: Zusammenfassung
